Data flow

When you send a chat message in the WebUI, it travels from your browser to hermes-webui (the chat frontend on port 8788), which forwards it as an API call to hermes-agent. hermes-agent selects the appropriate LLM, sends the request to the configured provider (e.g. OpenRouter, Anthropic, OpenAI), and streams the response back through hermes-webui to your browser.

Channel messages (Telegram, Slack, Discord) arrive at listeners running inside hermes-agent. Each listener receives an inbound message, passes it through the same LLM pipeline, and sends the response back via the channel's outbound API. hermes-station is not in this path — it only manages the gateway process lifecycle.

hermes-station owns the container orchestration layer: it starts and supervises hermes-webui and the hermes-agent gateway, exposes the admin UI, manages configuration files, and proxies all non-admin traffic to hermes-webui. hermes-agent owns the AI layer: model selection, MCP server connections, channel integrations, session memory, and tool execution. Configuration flows one way — hermes-station writes config.yaml and .env, and hermes-agent reads them on startup.
